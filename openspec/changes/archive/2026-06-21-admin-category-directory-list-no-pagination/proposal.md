## 为什么

当前 admin 侧的分类目录列表需要分页查看，导致运营或管理人员在需要整体扫视、批量核对或快速定位分类时频繁切页，降低了查看效率。这个变更的目标是让分类目录列表在 admin 场景下一次性展示全部条目，减少操作成本并提升可读性。

## 变更内容

- 取消 admin 侧分类目录列表的分页展示。
- 列表默认一次性加载并展示全部分类目录项。
- 保持现有列表内容、排序和交互语义不变，只调整展示方式。

## 能力范围

### 新增能力

- `admin-category-directory-list`: 管理后台分类目录列表支持全量展示，不再按页拆分。

### 修改能力

- `admin-status-badges`: 无变更，当前需求不涉及该能力的规范调整。

## 影响

- 受影响的 admin 列表页面展示逻辑。
- 受影响的后端查询/接口返回方式，可能需要支持一次返回完整列表。
- 当分类数量较大时，前端渲染与接口响应时间可能上升，需要在实现阶段评估性能边界。
